## 2. `src/server.ts`

### Exports
```ts
export async function startServer(dbPath: string, port: number): Promise<void>
```

### HTTP routes
- `GET /api/sessions` → `SessionResponse[]` sorted by updatedAt desc
- `GET /api/sessions/:id` → single `SessionResponse` or 404
- `GET /api/sessions/:id/events` → `EventResponse[]` sorted by timestamp asc

### Response shapes
```ts
interface SessionResponse {
  sessionId: string; startedAt: string; updatedAt: string; cwd: string;
  repoRoot: string|null; repoName: string|null; parentSessionId: string|null;
  gitUserName: string|null; gitUserEmail: string|null;
  prompt: string|null; summary: string|null;
  keywords: string[];  // parsed from JSON text
}
interface EventResponse {
  id: number; timestamp: string; event: string; sessionId: string;
  blocked: boolean;
  data: unknown;       // parsed JSON (full HookInput object)
  summary: string|null; keywords: string[];
}
```

### Mapping helpers (in server.ts)
`mapSession()`: `Date` → ISO string, parse `keywords` JSON text → `string[]`, null-coerce optionals
`mapEvent()`: same + `JSON.parse(e.data)`

### WebSocket change detection
Poll `db.event.findFirst({ orderBy: { id: 'desc' } })` every 2 s.
When max `id` changes, broadcast `{ type: "sessions_updated" }` to all OPEN clients.

### Other
- `cors()` middleware with no origin restriction (local dev tool)
- `WebSocketServer({ server: httpServer })` — shares port 3000 with HTTP
- SIGINT/SIGTERM → `clearInterval(poller); httpServer.close(); process.exit(0)`
- All local imports use `.js` extension (NodeNext requirement)

## 3. `handler.ts` change — add `serve` subcommand

Inside `main()`, after the `migrate` block, before the default stdin handler:
```ts
program
  .command("serve")
  .description("Start the HTTP + WebSocket API server.")
  .option("--db <file>", "SQLite database file", defaultDbPath)
  .option("--port <n>", "Port to listen on", (v: string) => parseInt(v, 10), 3000)
  .action(async (opts: { db: string; port: number }) => {
    const { startServer } = await import("./server.js");
    await startServer(expandHome(opts.db), opts.port);
  });
```
Dynamic import ensures express/ws don't load on every hook invocation.
`expandHome()` is called in handler.ts before passing to `startServer`, so `server.ts` receives an absolute path.
