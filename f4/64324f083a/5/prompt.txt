To understand the SubAgentStart and Stop events, look at the ../cli/CLAUDE.md in the entire project. The basic description of how it handles subagents calls is: 1. Agent-level hook parsing (per-agent lifecycle files)                                                                                     
                                                                                                                                              
  Each agent translates its native hooks into normalized SubagentStart/SubagentEnd events:                                                    
                  
  - Claude Code (cmd/entire/cli/agent/claudecode/lifecycle.go:153-185): maps PreTask → SubagentStart and PostTask → SubagentEnd. The PostTask 
  payload includes a tool_response.agentId field that becomes event.SubagentID — this is the spawned session's ID.
  - Cursor (cmd/entire/cli/agent/cursor/lifecycle.go:154-183): maps subagent-start/subagent-stop hooks.                                       
  - Factory AI Droid (cmd/entire/cli/agent/factoryaidroid/lifecycle.go:229-260): same pattern.                                                
  - Copilot CLI (cmd/entire/cli/agent/copilotcli/lifecycle.go:134): emits SubagentEnd on its subagentStop hook.                               
                                                                                                                                              
  The key linking field is event.ToolUseID — the tool_use_id from the parent's Task tool call, which ties the subagent's transcript to the    
  parent session.                                                                                                                             
                                                                                                                                              
  2. Normalized event dispatch (cmd/entire/cli/lifecycle.go:51-54)                                                                            
  
  case agent.SubagentStart:                                                                                                                   
      return handleLifecycleSubagentStart(...)                                                                                                
  case agent.SubagentEnd:
      return handleLifecycleSubagentEnd(...)                                                                                                  
                                                                                                                                              
  3. SubagentStart (lifecycle.go:631-647)
                                                                                                                                              
  Captures a pre-task filesystem snapshot keyed by ToolUseID via CapturePreTaskState. This baseline is used later to detect what the subagent 
  actually changed.
                                                                                                                                              
  4. SubagentEnd (lifecycle.go:649-779)                                                                                                       
  
  - Resolves the subagent's transcript path: <transcriptDir>/agent-<SubagentID>.jsonl                                                         
  - Extracts modified files from the subagent transcript (via TranscriptAnalyzer)
  - Diffs filesystem against the pre-task state                                                                                               
  - Calls strat.SaveTaskStep() which writes a checkpoint to the shadow branch at tasks/<tool_use_id>/ — the same path structure documented in
  CLAUDE.md
